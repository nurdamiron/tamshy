# Tamshy ↔ Qazsu

**Молодежный engagement layer над экосистемой управления водными ресурсами Казахстана**

Презентация для НАО «ИАЦ водных ресурсов» / Министерства водных ресурсов и ирригации
Дата: 2026-05-07
Контактное лицо: команда Tamshy

---

## Зачем

Qazsu решает учёт, мониторинг и управление водными ресурсами РК — 8 БВИ, 250+ ГТС, 334 гидропоста. Это **источник истины** для государства.

Tamshy решает массовое вовлечение школьников в водную повестку — конкурсы, проекты, экспертная оценка, всех 19 регионов.

Связка превращает официальные данные Qazsu в школьные проекты, а лучшие проекты — в публичную витрину молодежных решений на стороне министерства. Сквозной цикл:

```
Qazsu (контекст)  →  Tamshy (вовлечение)  →  Экспертная оценка  →  Витрина решений в Qazsu  →  Региональная аналитика
```

## Что получает министерство

### 1. Цифровизация учёта проектов
Каждая школьная работа теперь привязана к **региону, бассейну, водному объекту, типу проблемы**. Можно строить аналитику любых срезов.

### 2. Экологическое просвещение
Школы получают готовую проектную механику (регистрация → команда → загрузка → голосование → жюри → публикация) с привязкой к **реальному водному контексту страны**.

### 3. Молодежная инициатива (citizen science)
Конкретный канал, через который министерство получает идеи и наблюдения от школьников по конкретным водным объектам.

### 4. Публичная витрина решений
Раздел "Молодежные водные решения" — готов к встраиванию iframe'ом или прямыми запросами в `/api/qazsu/showcase`. Без ПД, только школа/регион/тема.

### 5. Региональная аналитика
KPI dashboard в админке: сколько проектов, по каким темам, в каких бассейнах, доля в витрине.

## Как это выглядит снаружи

### Сценарий A — Qazsu → Tamshy (deeplink)
1. Пользователь открывает карту Qazsu и выбирает водный объект (например, Иле).
2. Видит CTA "Создать школьный проект по этому объекту".
3. Переходит на `tamshy.kz/submit?source=qazsu&basin=BALKHASH_ALAKOL&water_object=ili-river&problem_type=WATER_QUALITY&ref_url=https://qazsu.gov.kz/#/object/ili-river`
4. Форма автоматически:
   - показывает badge "Сіз Qazsu платформасынан келдіңіз"
   - предзаполняет регион, бассейн, тему
   - сохраняет deeplink назад на объект Qazsu

### Сценарий B — Витрина в Qazsu
1. Министерство встраивает витрину `/api/qazsu/showcase` в свою экосистему.
2. На карте Qazsu появляется слой "молодежные решения" — кликая по бассейну, видно школьные проекты по этой территории.
3. Признанные проекты отображаются с пометкой "Победитель".

### Сценарий C — Аналитика
1. Кабинет ИАЦ заходит в `/admin/qazsu-stats`.
2. Видит барчарты: распределение проектов по 8 бассейнам, 11 темам, 19 регионам.
3. Видит топ-10 водных объектов (где школьники работают активнее).
4. Может фильтровать админку по бассейну/теме.

## Архитектура и безопасность

**3 уровня учётных данных в БД (PostgreSQL/RDS):**
- `prometric` — master, admin операции (CREATE ROLE, GRANT)
- `tamshy_migrator` — DDL для миграций (member of prometric)
- `tamshy` — runtime приложения (только DML)

**Что в витрину НЕ попадает (юр-блок):**
- ФИО учеников и учителей (ПД несовершеннолетних)
- Email и телефоны
- Полная команда

**Что попадает:**
- Название школы (публично-разрешено по соглашению Tamshy)
- Регион, бассейн, водный объект, тема
- Заголовок и аннотация проекта
- Обложка/видео (предоставленные авторами)

Каждый проект на момент подачи получает `qazsuSnapshot` (JSON с контекстом водного объекта на момент времени) — переименование объекта в будущем не сломает связь конкретной заявки с её темой.

## Что просим у министерства

### Вариант A — Лёгкая интеграция (нужно: ничего)
Tamshy продолжает работать как сейчас, периодически шлём отчёты по KPI.

### Вариант B — Партнёрство контентом (нужно: согласование 1-2 чел.)
- Министерство присылает CSV с дополнительными водными объектами (расширить с 65 до 250+ ГТС). Используем `/admin/water-objects` для импорта.
- Министерство шлёт пресс-релизы — заводим в News таблицу через `/admin/news-import`.
- В витрине Tamshy появляется "Партнёрский раздел Минводресурсов".

### Вариант C — Глубокая интеграция (нужно: API доступ + DNS)
- Qazsu добавляет CTA "Создать школьный проект" на карточках объектов с deeplink в Tamshy.
- Tamshy показывает "Открыть в Qazsu" из карточки своего проекта.
- Витрина `/api/qazsu/showcase` встраивается iframe'ом в Qazsu.
- Возможно: SSO между Qazsu и Tamshy (отдельный спринт).
